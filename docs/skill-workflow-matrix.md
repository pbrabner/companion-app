# Skill Workflow Matrix

Esta matriz complementa `docs/workflow-contract.md` e `docs/human-gates.md`.
Ela define o papel operacional das skills no ciclo de desenvolvimento.

## Matriz

| Skill | Papel | Estado de entrada | Estado de saida | Human gate | Evidencia |
|---|---|---|---|---|---|
| `start` | abrir contexto e checar saude | `intake` | `classified` ou `intake` pronto | nao, salvo bloqueio critico | `legion check`, handoff lido |
| `plan` | classificar alpha/beta/gamma e produzir plano/PRD/spec | `classified` | `planned` ou `approved` | sim para beta/gamma antes de executar | plano, PRD, criterios de aceite |
| `backend` | executar mudancas de API, DB, MCP, RAG | `approved` | `executing` | sim para schema/producao/secrets | diff, testes backend, migracoes dry-run |
| `frontend` | executar UI/UX/componentes | `approved` | `executing` | sim se publica/producao ou muda marca critica | diff, screenshot/teste visual quando cabivel |
| `data` | analise, SQL, metricas, modelos | `approved` | `executing` ou `verifying` | sim para dados destrutivos/custo/decisao executiva | query, amostra, validacao, premissas |
| `devops` | CI/CD, deploy, infra, observabilidade | `approved` | `executing` ou `human-review` | quase sempre em deploy/producao/secrets | plan/apply dry-run, logs, rollback |
| `qa` | revisar, testar, auditar, emitir verdict | `verifying` | `human-review` ou `closing` | sim se FAIL/ABSTAIN/risco alto | PASS/WARN/FAIL/ABSTAIN, comandos rodados |
| `end` | handoff, vault, aprendizados, commit/push | `closing` | `learned` | sim para commit/push sensivel ou release | handoff, vault updates, commit hash |
| `memory` | recall e persistencia de conhecimento | qualquer | mesmo estado ou `learned` | sim se conflito no vault | fonte, confianca, path da nota |

## Regras por Skill Core

### `start`

- Deve ler `docs/HANDOFF.md` antes de status/proxima acao.
- Deve usar `legion check` quando avaliar saude.
- Nao deve iniciar execucao de trabalho por conta propria.

### `plan`

- Deve classificar alpha/beta/gamma.
- Deve criar PRD/spec quando o escopo exigir.
- Deve parar antes de execucao se houver human gate.

### `qa`

- Deve separar PASS, WARN, FAIL e ABSTAIN.
- Deve reportar comandos e resultados numericos.
- Deve encaminhar ABSTAIN para `human-review`.

### `end`

- Deve fechar apenas com evidencia ou registrar pendencia.
- Deve manter `docs/HANDOFF.md` como status vivo.
- Nao deve fazer push/release sensivel sem aprovacao explicita.

## Skills de Dominio

Skills de dominio (`backend`, `frontend`, `data`, `devops`, `docs`,
`business`, `skills`, `vault-*`) executam dentro do escopo aprovado.
Quando encontrarem risco critico, devem parar e usar `docs/human-gates.md`.

## Fora de Escopo

Esta matriz nao reintroduz orchestrator, Ruflo ou skill-chains. Ela e um
contrato humano/operacional para Claude Code native + skills + hooks.
