# Human Gates

Este documento complementa `docs/workflow-contract.md`. Ele define quando o
Legion pode automatizar e quando deve parar para decisao humana.

## Regra Curta

```text
Automatize leitura, analise, teste, lint, dry-run e execucao reversivel dentro do escopo aprovado.
Pare para Andre em producao, secrets, delete, dados, custo, arquitetura e ABSTAIN.
```

## Dispatch behavior

Dispatch de skill (`legion workflow dispatch`) respeita esta tabela: risks em
SAFE_POLICY dispensam approval token; risks em CRITICAL_POLICY exigem
`--approval-id` valido. Detalhes em `docs/workflow-contract.md` secao
"Dispatch Layer".

## Gate Obrigatorio

| Categoria | Exemplos | Motivo |
|---|---|---|
| producao | cutover, deploy, rollback, pm2 restart de servico vivo | impacto real em usuario/sistema |
| deploy | Vercel, Supabase, PM2, GitHub release | pode publicar estado ruim |
| secrets | `.env`, tokens, auth, service role, OAuth | risco de vazamento ou lockout |
| delete | remover arquivos, tabelas, dados ou skills | pode ser irreversivel |
| migracao destrutiva | drop/truncate/delete/update em massa | perda de dados |
| custo | billing, quota, plano pago, API externa cara | impacto financeiro |
| arquitetura | troca de stack, contrato publico, storage, auth | dificil de reverter |
| conflito no vault | fontes contraditorias ou baixa confianca | memoria errada vira regra futura |
| ABSTAIN | Auditor, Oracle, Sentinel ou QA sem evidencia | ausencia de certeza nao e aprovacao |
| PRD beta/gamma | tarefa media/grande antes de executar | evita one-shotting |

## Pode Automatizar

| Categoria | Exemplos | Condicao |
|---|---|---|
| leitura | `git status`, `rg`, `Get-Content`, inventario | sem exfiltrar secrets |
| busca | vault lookup, code search, docs search | read-only |
| analise | diff review, triagem, classificacao | sem aplicar mudanca |
| testes | pytest, vitest, smoke, lint | comandos nao destrutivos |
| lint | ruff, mypy, format check | dentro do repo |
| dry-run | plano, proposta, preview, audit | nao escreve ou escreve apenas report |
| docs nao vivos | docs historicos ou reports datados | nao substituir `docs/HANDOFF.md` |
| codigo local | mudanca reversivel dentro de escopo aprovado | com teste/verificacao |

## Escalada

Se qualquer item cair em duas categorias, vence a mais restritiva.

Exemplo:

```text
Alterar script de deploy + rodar teste local = human gate, porque deploy vence teste.
```

## Formato de Pedido ao Andre

Quando parar, apresentar:

- decisao necessaria;
- risco;
- opcoes;
- recomendacao;
- comando ou arquivo que sera afetado;
- criterio para prosseguir.

Nao usar pergunta vaga. A aprovacao deve ser explicita.
